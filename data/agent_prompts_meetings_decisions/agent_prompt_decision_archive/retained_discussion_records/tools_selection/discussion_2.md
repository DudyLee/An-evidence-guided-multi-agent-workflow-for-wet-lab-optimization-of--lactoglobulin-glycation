Principal Investigator: ### Agenda

## Final decision

### 1) Core project logic

The project should be governed by a **shortest-practical-strong-window rule**:

- Prefer the **earliest** condition that:
  1. reaches **>=80% reduction**, or
  2. reaches a practical plateau where an added hour gives **<10 percentage points** additional reduction.
- A later condition should only be preferred if it gives a **clearly meaningful gain** that justifies:
  - extra time,
  - extra byproduct risk,
  - extra temperature burden,
  - or extra process complexity.

This means the project should optimize for the **efficient frontier**, not the numerical maximum.

#### Practical ranking rule
Use this simple ranking order for matched conditions:
1. **Band A:** >=80%
2. **Band B:** 70–79% if meaningfully faster, milder, or simpler
3. Prefer the **earlier timepoint**
4. Reject extra time if gain is **<10 percentage points per added hour**
5. If still close, prefer:
   - lower temperature,
   - lower ultrasound dependence,
   - cleaner interpretability

---

### 2) Donor priority logic

Use a **matched four-donor core panel**:

#### Benchmark donors
- **lactose**
- **glucose**

#### Challenger donors
- **galactose**
- **mannose**

**Defer pentoses**, including arabinose, from the immediate next phase.

Rationale:
- lactose and glucose are the strongest current local wet-lab benchmarks,
- galactose and mannose are the highest-value literature-supported untested challengers,
- pentoses are more likely to accelerate later-stage chemistry and increase byproduct burden,
- and current local evidence already shows **arabinose underperforming lactose and glucose** in the tested domain.

This is enough evidence to justify **deferral**, though not permanent exclusion.

---

### 3) Reaction mode, temperature, pH, and pretreatment logic

#### Reaction mode
- Keep **dry-state glycation** as the primary next-phase platform.

#### Temperature
- Focus the immediate next phase on **55–60 C**.

##### Why these two temperatures
- **60 C**
  - stronger kinetic anchor
  - more likely to reveal donor separation in a short window
- **55 C**
  - lower thermal burden
  - potentially better control and cleaner interpretability

So:
- **60 C tests short-window activity**
- **55 C tests whether similar performance can be achieved more gently**

#### pH
- **Defer pH** from the immediate dry bridge phase.
- If later tested, start with:
  - **neutral vs mildly alkaline**
  - around **pH 7.5–8.0**
- Do **not** prioritize acidic conditions now.

#### Pretreatment
- Use **ultrasound as the default main pretreatment arm**
- but do **not** assume it is universally necessary.

##### Ultrasound classification rule
Call ultrasound:

- **Necessary** if removing it causes the branch to:
  - fall below Band B,
  - or require a harsher condition such as:
    - +1 h extra time,
    - or higher temperature

- **Practically enabling** if removing it keeps the branch active but clearly worsens efficiency or severity

- **Marginally beneficial** if removing it changes the number slightly but does not change practical ranking

This is the simplest operational rule that still answers the agenda question.

---

### 4) Strong practical candidate definition

Adopt a **two-stage evidence model**.

#### Stage A: provisional hit
A condition may be considered a provisional hit based on strong reported reduction in a proper matched comparison.

#### Stage B: promoted strong practical candidate
A condition should only be promoted if it satisfies both:

##### Efficacy / practicality criteria
- ideally **>=80% reduction**
- achieved in the **shortest practical time**, preferably within **2–4 h**
- within **55–60 C**
- without unnecessary severity or complexity
- and consistent with the **<10 percentage point per added hour** rule

##### Minimum interpretability gate
- **matched heated no-sugar control**
- **acceptable extractability / recovery**
- **basic aggregation / size-state assessment**
- **no dominant insolubilization artifact**
- **one orthogonal immunological confirmation** before final winner declaration

##### Practical checklist for a strong candidate
A condition should count as strong only if the answer is effectively **yes** to all of the following:
- Is reduction in **Band A**, or in **Band B with a real burden advantage**?
- Is it at or near the **earliest practical plateau**?
- Is temperature no harsher than needed?
- Is ultrasound no more necessary than needed?
- Does it pass the minimum interpretability gate?

##### What counts as unnecessary severity or complexity
Examples:
- 4 h instead of 3 h for only a trivial gain
- 60 C instead of 55 C with no clear practical benefit
- ultrasound dependence when no-ultrasound remains competitive
- strong apparent reduction with poor recovery or major aggregation
- adding pH manipulation before the donor/time/temperature question is resolved

---

### 5) Next-phase structure

The next phase should be a **bridge phase**, not broad optimization.

Its purpose should be to answer:

1. Does donor ranking change outside the current local domain?
2. Is **55 C** competitive with or preferable to **60 C**?
3. Does **4 h** really earn its place?
4. Is ultrasound **necessary**, **enabling**, or only **marginally beneficial**?
5. Which donor branch deserves time-refinement next?

## Specific next-phase recommendation

### Preferred structure
Use a **compact matched dry-state donor bridge** with:
- donors:
  - lactose
  - glucose
  - galactose
  - mannose
- temperatures:
  - **55 C**
  - **60 C**
- pretreatment:
  - **ultrasound main arm**
  - **selected no-ultrasound controls**
- interpretation safeguards:
  - at least one **heated no-sugar control**
  - aggregation / recovery checks on leading branches

### Time strategy
Use a staged rule:

#### Protected bridge screen
- run the full donor bridge at **4 h** across **55 C and 60 C**

#### Early-time follow-up
- run **3 h follow-up** for the **top 2 donor branches**
- optionally run **2 h follow-up** for the **top 1 branch**

This remains the best compromise because it keeps the bridge compact while still testing whether 4 h is actually needed.

### Simplified contingency logic
Keep only the most likely contingencies:

- **If one donor is clearly ahead and interpretation-clean:**  
  advance it.

- **If two donors are close or differ on practical strengths:**  
  advance both.

- **If donors cluster tightly:**  
  do not force a winner; check reproducibility and refine time first.

- **If 55 C and 60 C are similar:**  
  prefer **55 C**.

- **If the top numerical condition fails interpretability checks:**  
  do not promote it.

- **If only Band B conditions emerge:**  
  retain the best shorter/cleaner Band B branches rather than forcing a weak Band A narrative.

### Post-bridge advancement rule
- **Advance one donor only** if it:
  - reaches **>=80%**
  - is about **10 percentage points** ahead of the next-best donor
  - is stable across the matched context
  - and passes the interpretability gate

- **Advance two donors** if:
  - the top two are both strong and within **<10 points**, or
  - one is slightly lower numerically but clearly better in efficiency or interpretability

- **Keep conclusions provisional** if:
  - donor rankings cluster,
  - rankings shift across temperature,
  - or the apparent winner fails interpretability checks

## 5. How to convert glycation maps into a biologically meaningful score

I strongly support the chemistry scientist’s idea that the score should be **distance-weighted**, not binary. From the immunology side, I would refine that further.

### A useful score should consider at least four components

#### A. Epitope overlap score
Modification probability at residues directly within known/predicted epitope regions.

#### B. Epitope adjacency score
Modification probability within a defined spatial shell around epitope residues, since nearby adducts can sterically hinder antibody binding even without direct residue overlap.

#### C. Accessibility change score
Whether the pretreatment + glycation combination reduces solvent exposure of epitope surfaces or places bulky adducts in their approach path.

#### D. Specificity score
How much of the predicted modification burden is concentrated in epitope-relevant regions versus spread diffusely over irrelevant areas.

That last term is very important. A condition that predicts 12 possible modifications all over the surface may look “strong” numerically, but immunologically it may be less attractive than a condition that predicts 3–4 focused modifications in high-value epitope neighborhoods.

### My recommendation for scoring philosophy
Use a **benefit-focused score with a specificity penalty**, not a bulk-coverage score.

**OTP(s)**

## 2. How I would define distance weighting

I recommend a **three-tier practical scheme**, not a continuous fancy function. Simpler is better here.

### Tier 1: Direct overlap
If the Lys residue lies **within a known IgE epitope region**, assign the highest weight.

### Tier 2: Near-epitope adjacency
If the Lys residue is **adjacent in sequence** to a linear epitope region or lies within a **local spatial shell** around epitope residues in the 3D structure, assign an intermediate weight.

### Tier 3: Distal
If the Lys residue is outside both, assign low or zero weight.

### Concrete implementation
For structure-based weighting, I would use:

- **Direct epitope-associated**: Lys side chain within about **0–5 Å** of any epitope residue surface region
- **Adjacent / neighborhood-associated**: about **5–10 Å**
- **Distal**: **>10 Å**

This is not because 5 and 10 Å are magic immunological constants, but because they are practical bins for:
- direct local steric interference,
- nearby shielding/conformational perturbation,
- and weakly relevant distant modification

For a simple normalized scheme:
- **Direct overlap / 0–5 Å**: 1.0
- **Adjacent / 5–10 Å**: 0.5
- **Distal / >10 Å**: 0.0 to 0.1

My recommendation is to adopt a **compact, ensemble-based, literature-anchored virtual triage pipeline** with the following **final core stack**:

- and **no literal multi-sugar clustering model** in the main pipeline.

#### Final decision

We will define the computational objective as:

> **A virtual filter that ranks glycation candidates by predicted epitope-interference opportunity versus predicted modification burden/interpretation risk, in order to prioritize wet-lab branches.**

We will **not** use the term “Allergenicity Reduction Score” as the primary label.  
Instead, the pipeline will output:

- **Predicted Epitope Interference Benefit (PEIB)**
- **Predicted Modification Burden / Interpretation Risk**

plus supporting submetrics:
- Direct Epitope Hit Fraction
- Epitope-Focused Opportunity Fraction
- Pretreatment Gain

#### Why this recommendation

This choice best satisfies all constraints from the meeting:

- It is **small enough to run in practice**.
- It is **chemically honest**, because it does not pretend docking proves glycation chemistry.
- It is **biologically grounded**, because it prioritizes known β-LG epitopes over generic prediction software.
- It is **decision-linked**, because it produces a shortlist and recommendation classes.
- It is **future-proof**, because it can compare arabinose/tagatose now but extend to a broader donor panel later.

#### Operational ranking rule

Use this ranking order:

1. Prefer higher **PEIB**
2. Do not promote candidates with:
   - dominant off-epitope spread,
   - benefit dependent mainly on upper-bound perturbed conformers,
   - or no meaningful pretreatment gain
3. Break ties by:
   - lower Predicted Modification Burden / Interpretation Risk,
   - lower pretreatment dependence,
   - lower sugar chemistry caution

#### Stop rules

Use these stop rules:

- **Pretreatment not justified** if mild perturbation increases PEIB by **<10–15%** and does not improve epitope focus category.
- **Sugars tied** if PEIB differs by **<10%** and burden class is the same.
- **Diffuse branch** if **>60%** of predicted opportunity is off-epitope.
- **Downgrade for instability** if advantage appears mainly in upper-bound perturbed ensemble.
- **Drop BepiPred/DiscoTope from core** if they do not change ranking.

#### Recommendation classes

Assign each branch to one of:

- **Advance**
- **Advance with caution**
- **Reserve**
- **Deprioritize**

This is the most actionable endpoint for wet-lab planning.

### Answers

#### 1. Which tools can accurately predict **Lysine reactivity changes** induced by our proposed pretreatments (e.g., partial unfolding)?

**Answer:**  
Use an **ensemble-based structural analysis core** consisting of:
- **experimental β-LG A structure + conformer generation**
- **FreeSASA**
- **PROPKA**

Treat **NetGlycate** only as optional annotation, not a core tool.

**Justification:**  
The team agreed that pretreatment effects should be represented by comparing native and mildly perturbed ensembles, not by direct ultrasound simulation. Lys reactivity is best approximated by combining:
- solvent exposure,
- ensemble persistence of exposure,
- and local protonation/reactivity context.

This is more defensible than SASA alone or sequence-only glycation prediction.

#### 2. For our chosen sugars (e.g., Arabinose), which docking tools are best suited to simulate **multi-site glycation** and sugar clustering on the protein surface?

**Answer:**  
Use **AutoDock Vina** as the core docking tool for **repeated, site-focused ensemble docking** near top Lys neighborhoods.  
Do **not** model multi-site glycation as a literal stable multi-sugar complex in the core workflow.

**Justification:**  
The team agreed that “multi-site glycation” should be operationalized as an **aggregated site-opportunity map**, not as an atomically realistic multi-sugar bound state. AutoDock Vina is the best fit because it is:
- lightweight,
- scalable across multiple conformers and sites,
- and adequate for repeated local docking of small sugars.

HADDOCK may be used only as optional refinement, not as core.

#### 3. How can we combine structural models with epitope prediction tools (like BepiPred) to create a quantitative **'Allergenicity Reduction Score'**?

**Answer:**  
We should **not** use “Allergenicity Reduction Score” as the primary term.  
Instead, calculate a **Predicted Epitope Interference Benefit (PEIB)**:

\[
PEIB(c)=\sum_i [LOS_i(c)\times SES_i(c)\times E_i]
\]

where:
- \(LOS_i\) = Lys Opportunity Score from ensemble exposure + pKa/context
- \(SES_i\) = Sugar Encounter Score from repeated local docking
- \(E_i\) = epitope relevance weight based primarily on **curated known β-LG epitopes**, with optional BepiPred/DiscoTope support only if decision-changing

Use a simple epitope weighting scheme:
- **0–5 Å / direct** = high weight
- **5–10 Å / adjacent** = intermediate weight
- **>10 Å / distal** = low or zero weight

Pair PEIB with a second axis:
- **Predicted Modification Burden / Interpretation Risk**

**Justification:**  
The team agreed that the model cannot directly predict allergenicity reduction in the full immunological sense. PEIB is a more honest and useful proxy because it integrates:
- site reactivity,
- sugar encounter plausibility,
- and epitope relevance.

The burden axis prevents overpromotion of diffuse or chemically aggressive branches.

#### 4. Propose a specific **In Silico Pipeline** (Step 1 -> Step 2 -> Step 3) that transforms our Theoretical SOP into a refined list of wet-lab candidates.

**Answer:**  

### Step 1: Build structural ensembles
- Start from experimental β-LG A structure
- Generate:
  - native ensemble
  - mild pretreatment-surrogate ensemble
  - optional upper-bound perturbed ensemble for sensitivity only

### Step 2: Map Lys opportunity
- Use **FreeSASA** to quantify Lys exposure across conformers
- Use **PROPKA** to estimate local protonation/reactivity context
- Generate per-Lys **Lys Opportunity Scores (LOS)**

### Step 3: Map sugar encounter opportunity
- Select top Lys neighborhoods
- Use **AutoDock Vina** for repeated local docking of each sugar across representative conformers
- Generate per-site **Sugar Encounter Scores (SES)**

### Step 4: Overlay epitope relevance
- Map curated known β-LG epitopes onto the structure
- Optionally add BepiPred/DiscoTope only if they change ranking
- Compute epitope relevance weights and PEIB

### Step 5: Rank and classify candidates
For each candidate branch:
- compute **PEIB**
- compute **Predicted Modification Burden / Interpretation Risk**
- compute:
  - Direct Epitope Hit Fraction
  - Epitope-Focused Opportunity Fraction
  - Pretreatment Gain
- assign recommendation class:
  - Advance
  - Advance with caution
  - Reserve
  - Deprioritize

Output:
- a **2D benefit-vs-burden plot**
- and a **ranked decision table**
- yielding a refined shortlist of wet-lab candidates

**Justification:**  
This is the smallest complete workflow that satisfies the agenda’s three required categories:
- reactivity prediction,
- molecular interaction,
- epitope analysis.

It also directly produces the required output: a **Recommended Experimental Parameter Set** in the form of ranked candidate branches and recommendation classes.
