Principal Investigator: ### Agenda

## Final decision

### 1) Core project logic

The project should be governed by a **shortest-practical-strong-window rule**:

- Prefer the **earliest** condition that:
  1. reaches **>=80% reduction**, or
  2. reaches a practical plateau where an added hour gives **<10 percentage points** additional reduction.
- A later condition should only be preferred if it gives a **clearly meaningful gain** that justifies:
  - extra time,
  - extra byproduct risk,
  - extra temperature burden,
  - or extra process complexity.

This means the project should optimize for the **efficient frontier**, not the numerical maximum.

#### Practical ranking rule
Use this simple ranking order for matched conditions:
1. **Band A:** >=80%
2. **Band B:** 70–79% if meaningfully faster, milder, or simpler
3. Prefer the **earlier timepoint**
4. Reject extra time if gain is **<10 percentage points per added hour**
5. If still close, prefer:
   - lower temperature,
   - lower ultrasound dependence,
   - cleaner interpretability

---

### 2) Donor priority logic

Use a **matched four-donor core panel**:

#### Benchmark donors
- **lactose**
- **glucose**

#### Challenger donors
- **galactose**
- **mannose**

**Defer pentoses**, including arabinose, from the immediate next phase.

Rationale:
- lactose and glucose are the strongest current local wet-lab benchmarks,
- galactose and mannose are the highest-value literature-supported untested challengers,
- pentoses are more likely to accelerate later-stage chemistry and increase byproduct burden,
- and current local evidence already shows **arabinose underperforming lactose and glucose** in the tested domain.

This is enough evidence to justify **deferral**, though not permanent exclusion.

---

### 3) Reaction mode, temperature, pH, and pretreatment logic

#### Reaction mode
- Keep **dry-state glycation** as the primary next-phase platform.

#### Temperature
- Focus the immediate next phase on **55–60 C**.

##### Why these two temperatures
- **60 C**
  - stronger kinetic anchor
  - more likely to reveal donor separation in a short window
- **55 C**
  - lower thermal burden
  - potentially better control and cleaner interpretability

So:
- **60 C tests short-window activity**
- **55 C tests whether similar performance can be achieved more gently**

#### pH
- **Defer pH** from the immediate dry bridge phase.
- If later tested, start with:
  - **neutral vs mildly alkaline**
  - around **pH 7.5–8.0**
- Do **not** prioritize acidic conditions now.

#### Pretreatment
- Use **ultrasound as the default main pretreatment arm**
- but do **not** assume it is universally necessary.

##### Ultrasound classification rule
Call ultrasound:

- **Necessary** if removing it causes the branch to:
  - fall below Band B,
  - or require a harsher condition such as:
    - +1 h extra time,
    - or higher temperature

- **Practically enabling** if removing it keeps the branch active but clearly worsens efficiency or severity

- **Marginally beneficial** if removing it changes the number slightly but does not change practical ranking

This is the simplest operational rule that still answers the agenda question.

---

### 4) Strong practical candidate definition

Adopt a **two-stage evidence model**.

#### Stage A: provisional hit
A condition may be considered a provisional hit based on strong reported reduction in a proper matched comparison.

#### Stage B: promoted strong practical candidate
A condition should only be promoted if it satisfies both:

##### Efficacy / practicality criteria
- ideally **>=80% reduction**
- achieved in the **shortest practical time**, preferably within **2–4 h**
- within **55–60 C**
- without unnecessary severity or complexity
- and consistent with the **<10 percentage point per added hour** rule

##### Minimum interpretability gate
- **matched heated no-sugar control**
- **acceptable extractability / recovery**
- **basic aggregation / size-state assessment**
- **no dominant insolubilization artifact**
- **one orthogonal immunological confirmation** before final winner declaration

##### Practical checklist for a strong candidate
A condition should count as strong only if the answer is effectively **yes** to all of the following:
- Is reduction in **Band A**, or in **Band B with a real burden advantage**?
- Is it at or near the **earliest practical plateau**?
- Is temperature no harsher than needed?
- Is ultrasound no more necessary than needed?
- Does it pass the minimum interpretability gate?

##### What counts as unnecessary severity or complexity
Examples:
- 4 h instead of 3 h for only a trivial gain
- 60 C instead of 55 C with no clear practical benefit
- ultrasound dependence when no-ultrasound remains competitive
- strong apparent reduction with poor recovery or major aggregation
- adding pH manipulation before the donor/time/temperature question is resolved

---

### 5) Next-phase structure

The next phase should be a **bridge phase**, not broad optimization.

Its purpose should be to answer:

1. Does donor ranking change outside the current local domain?
2. Is **55 C** competitive with or preferable to **60 C**?
3. Does **4 h** really earn its place?
4. Is ultrasound **necessary**, **enabling**, or only **marginally beneficial**?
5. Which donor branch deserves time-refinement next?

## Specific next-phase recommendation

### Preferred structure
Use a **compact matched dry-state donor bridge** with:
- donors:
  - lactose
  - glucose
  - galactose
  - mannose
- temperatures:
  - **55 C**
  - **60 C**
- pretreatment:
  - **ultrasound main arm**
  - **selected no-ultrasound controls**
- interpretation safeguards:
  - at least one **heated no-sugar control**
  - aggregation / recovery checks on leading branches

### Time strategy
Use a staged rule:

#### Protected bridge screen
- run the full donor bridge at **4 h** across **55 C and 60 C**

#### Early-time follow-up
- run **3 h follow-up** for the **top 2 donor branches**
- optionally run **2 h follow-up** for the **top 1 branch**

This remains the best compromise because it keeps the bridge compact while still testing whether 4 h is actually needed.

### Simplified contingency logic
Keep only the most likely contingencies:

- **If one donor is clearly ahead and interpretation-clean:**  
  advance it.

- **If two donors are close or differ on practical strengths:**  
  advance both.

- **If donors cluster tightly:**  
  do not force a winner; check reproducibility and refine time first.

- **If 55 C and 60 C are similar:**  
  prefer **55 C**.

- **If the top numerical condition fails interpretability checks:**  
  do not promote it.

- **If only Band B conditions emerge:**  
  retain the best shorter/cleaner Band B branches rather than forcing a weak Band A narrative.

### Post-bridge advancement rule
- **Advance one donor only** if it:
  - reaches **>=80%**
  - is about **10 percentage points** ahead of the next-best donor
  - is stable across the matched context
  - and passes the interpretability gate

- **Advance two donors** if:
  - the top two are both strong and within **<10 points**, or
  - one is slightly lower numerically but clearly better in efficiency or interpretability

- **Keep conclusions provisional** if:
  - donor rankings cluster,
  - rankings shift across temperature,
  - or the apparent winner fails interpretability checks

## My current instinct on tool philosophy

## 1. What would make an in silico epitope prediction biologically credible?

For me, credibility requires four things.

### A. It must be anchored to **known β-LG epitope evidence**
Not just generic B-cell epitope prediction. β-LG has a literature on:
- linear IgE-reactive segments,
- conformational surface regions,
- and patient-serum-dependent recognition heterogeneity.

So the pipeline should start with a **curated β-LG epitope map**, then use prediction tools only to extend or contextualize that map.

If we rely only on BepiPred/DiscoTope de novo outputs, we risk building on tool priors rather than on β-LG-specific immunology.

### B. It must distinguish **epitope-disruptive** from **epitope-adjacent but irrelevant** modification
A Lys near an epitope is not automatically useful. The modification has to plausibly:
- alter a residue directly involved in antibody recognition,
- obstruct access to a critical patch,
- or shift local conformation enough to reduce recognition.

Merely being “within X angstroms” is too weak by itself.

### C. It must consider **conformational epitopes**, not just linear ones
β-LG allergenicity is not reducible to linear sequence segments alone. Many assay readouts are influenced by the preserved or altered 3D fold. So if the pipeline only scores linear overlap, it will miss important biology and may overvalue modifications in flexible loops that are not major conformational determinants.

### D. It must penalize scenarios likely to create **assay-specific signal loss without true immunological improvement**
This is crucial. In wet lab, lower ELISA or immunoblot signal can come from:
- aggregation,
- insolubilization,
- altered adsorption to plates,
- extraction loss,
- or steric masking in one assay format only.

In silico we cannot fully model those artifacts, but we can at least avoid scores that reward:
- heavily clustered surface modification likely to drive aggregation,
- modifications far from validated epitope regions,
- or broad nonspecific surface coating that may reduce antibody access without necessarily reducing biological allergenicity.

## 3. What should anchor the score?

I would structure the score around **epitope-relevant modification opportunity**, not just total glycation opportunity.

### Core principle
A donor/pretreatment condition should rank well only if it preferentially increases modification plausibility at **epitope-relevant, surface-accessible, immunologically important regions**.

A condition that predicts many modifications on β-LG but mostly outside known IgE-relevant regions should not score highly.

### Proposed score components

#### 1. **Epitope-Weighted Site Reactivity**
For each Lys (and possibly N-terminus):
- predicted exposure
- change in exposure after pretreatment
- local chemistry favorability
- multiplied by an **epitope relevance weight**

This is the most important term.

#### 2. **Predicted Surface Interference**
For each plausible glycation site:
- does the attached sugar likely occupy space overlapping a known linear or conformational epitope patch?
- does it alter local electrostatic/hydrogen-bonding environment in that patch?

This is where docking or local adduct placement can help, but it must remain local and comparative.

#### 3. **Conformational Epitope Preservation/Disruption Context**
If a candidate modification pattern is likely to destabilize or reshape an epitope-bearing region, that may matter more than pure steric masking. Conversely, if the global fold remains essentially unchanged and the adduct is peripheral, the effect may be minor.

This term should be modestly weighted because prediction uncertainty is high.

#### 4. **Artifact Risk Penalty**
Apply penalties for patterns likely to create misleading assay signal loss rather than meaningful reduction, such as:
- dense modification clustering on non-epitope regions,
- likely aggregation-prone patching,
- dominant modification of regions associated more with solubility changes than epitope interference,
- or excessive dependence on a single modeled structural state

This is one place where I slightly expand on the chemistry view: I think the artifact penalty must be explicit, not implicit.

#### 5. **Coverage Across Epitope Classes**
A strong candidate should ideally affect more than one relevant epitope class:
- one linear hotspot only may be too fragile a gain,
- broader but still targeted conformational interference is more compelling.

That said, we should not reward indiscriminate coating.

## 1. What would make an in silico epitope prediction biologically credible?

For me, credibility requires four things.

### A. It must be anchored to **known β-LG epitope evidence**
Not just generic B-cell epitope prediction. β-LG has a literature on:
- linear IgE-reactive segments,
- conformational surface regions,
- and patient-serum-dependent recognition heterogeneity.

So the pipeline should start with a **curated β-LG epitope map**, then use prediction tools only to extend or contextualize that map.

If we rely only on BepiPred/DiscoTope de novo outputs, we risk building on tool priors rather than on β-LG-specific immunology.

### B. It must distinguish **epitope-disruptive** from **epitope-adjacent but irrelevant** modification
A Lys near an epitope is not automatically useful. The modification has to plausibly:
- alter a residue directly involved in antibody recognition,
- obstruct access to a critical patch,
- or shift local conformation enough to reduce recognition.

Merely being “within X angstroms” is too weak by itself.

### C. It must consider **conformational epitopes**, not just linear ones
β-LG allergenicity is not reducible to linear sequence segments alone. Many assay readouts are influenced by the preserved or altered 3D fold. So if the pipeline only scores linear overlap, it will miss important biology and may overvalue modifications in flexible loops that are not major conformational determinants.

### D. It must penalize scenarios likely to create **assay-specific signal loss without true immunological improvement**
This is crucial. In wet lab, lower ELISA or immunoblot signal can come from:
- aggregation,
- insolubilization,
- altered adsorption to plates,
- extraction loss,
- or steric masking in one assay format only.

In silico we cannot fully model those artifacts, but we can at least avoid scores that reward:
- heavily clustered surface modification likely to drive aggregation,
- modifications far from validated epitope regions,
- or broad nonspecific surface coating that may reduce antibody access without necessarily reducing biological allergenicity.

## 3. What should anchor the score?

I would structure the score around **epitope-relevant modification opportunity**, not just total glycation opportunity.

### Core principle
A donor/pretreatment condition should rank well only if it preferentially increases modification plausibility at **epitope-relevant, surface-accessible, immunologically important regions**.

A condition that predicts many modifications on β-LG but mostly outside known IgE-relevant regions should not score highly.

### Proposed score components

#### 1. **Epitope-Weighted Site Reactivity**
For each Lys (and possibly N-terminus):
- predicted exposure
- change in exposure after pretreatment
- local chemistry favorability
- multiplied by an **epitope relevance weight**

This is the most important term.

#### 2. **Predicted Surface Interference**
For each plausible glycation site:
- does the attached sugar likely occupy space overlapping a known linear or conformational epitope patch?
- does it alter local electrostatic/hydrogen-bonding environment in that patch?

This is where docking or local adduct placement can help, but it must remain local and comparative.

#### 3. **Conformational Epitope Preservation/Disruption Context**
If a candidate modification pattern is likely to destabilize or reshape an epitope-bearing region, that may matter more than pure steric masking. Conversely, if the global fold remains essentially unchanged and the adduct is peripheral, the effect may be minor.

This term should be modestly weighted because prediction uncertainty is high.

#### 4. **Artifact Risk Penalty**
Apply penalties for patterns likely to create misleading assay signal loss rather than meaningful reduction, such as:
- dense modification clustering on non-epitope regions,
- likely aggregation-prone patching,
- dominant modification of regions associated more with solubility changes than epitope interference,
- or excessive dependence on a single modeled structural state

This is one place where I slightly expand on the chemistry view: I think the artifact penalty must be explicit, not implicit.

#### 5. **Coverage Across Epitope Classes**
A strong candidate should ideally affect more than one relevant epitope class:
- one linear hotspot only may be too fragile a gain,
- broader but still targeted conformational interference is more compelling.

That said, we should not reward indiscriminate coating.

## 6. How to combine outputs without overfitting

- ΔSASA after pretreatment
- local pKa favorability

Based on:
- overlap with curated high-confidence epitope regions
- support from BepiPred / DiscoTope
- number of epitope-relevant sites affected

#### C. **Targeted Interference Index**
Based on:

#### D. **Artifact Risk Index**
Based on:
- broad non-epitope surface modification
- aggregation-prone clustering proxy
- dependence on highly perturbed / low-confidence conformers

- prioritize high **Epitope Relevance**
- then high **Reactivity Opportunity**
- then high **Targeted Interference**
- while rejecting high **Artifact Risk**

- the component scores separately

## 2. A simple donor-reactivity prior that does not overreward aggressive chemistry

Yes. I think we need a **bounded donor prior**, not a dominant one.

The purpose is not to declare “most reactive donor wins.”  
The purpose is to reflect that some donors are more likely than others to engage in early glycation under short moderate dry conditions.

### Proposed principle
Use a **moderate donor reactivity prior** on a normalized 0–1 scale, with a narrow range so it cannot overpower site opportunity and epitope relevance.

For the **core donor panel**, I would use something like:

- **glucose: 0.65**
- **galactose: 0.60**
- **mannose: 0.55**
- **lactose: 0.40**

I am deliberately keeping the spread modest.

### Why this ordering?
Without pretending to excessive precision:

- **glucose**: reasonable baseline practical reactivity in early glycation
- **galactose**: similar class, potentially comparable but not assumed superior
- **mannose**: plausible challenger, but I would not inflate it without matched evidence
- **lactose**: lower intrinsic reactivity as a bulkier disaccharide, but still practically important because steric masking geometry and local selectivity may compensate

### Important rule
This prior should be:
- **low weight**
- and ideally **winsorized / capped**

For example, donor prior should maybe contribute only **10–15%** of a donor plausibility component, not 40–50%.

### Overprocessing penalty
I also recommend a separate **overprocessing-risk modifier**, but only for exploratory donors more likely to run fast into later-stage chemistry.

For the core hexose panel, I would keep this penalty very light or even neutral unless there is a specific donor reason to apply it.

For exploratory donors:
- **arabinose** would receive a stronger caution penalty
- **tagatose** might also warrant caution depending on how the team chooses to position it

### Bottom line
The donor prior should:
- gently separate donors,
- not dominate ranking,
- and never override strong epitope-targeted site opportunity.

## 3. Minimum acceptable representation of pretreatment-induced partial unfolding if full MD is too heavy

**Short restrained MD / short elevated-temperature relaxation / limited ensemble sampling**

- enough to compare native vs mildly perturbed states
- not enough to claim realistic ultrasound simulation

### Acceptable fallback
**Normal mode analysis or elastic-network perturbation**
- generate low-frequency motion conformers
- identify regions where small perturbation increases Lys exposure
- compare ΔSASA and epitope-patch exposure across the ensemble

### Bare-minimum fallback
If resources are very limited:
- use the native structure,
- computationally perturb flexible loops / partially buried regions near candidate Lys,
- generate a **small conformer panel**,

- using only the static native structure and calling that pretreatment-aware
- or applying a generic “ultrasound exposure bonus” with no structural basis

### Chemistry reason
Pretreatment matters because glycation in our window is likely controlled by exposure of a few key sites. If we do not model exposure change at all, we are not really addressing the pretreatment question.
